# 新版航電遙測通訊格式規範 (Telemetry Format Specification)

本文件定義了新版航電板與地面站 (Ground Side) 之間，透過串口 (Serial) 或無線 LoRa 傳輸數據的 ASCII 通訊格式規範。

## 通訊協議特點

1.  **分開字元**：每個數據欄位之間使用**一個半形空格**進行分隔。
2.  **結束字元**：整行數據以 `\r\n` (CRLF) 作為結束符號。
3.  **高容錯解析**：每個欄位都具備明確的英文大寫 Key 前綴，解析器會動態提取對應的數值。即便傳輸過程中發生局部丟包（欄位缺失），解析器仍能正確還原剩餘的有效數據，不會造成整幀失效。

---

## 遙測數據格式範例

### 完整 ASCII 遙測字串 (無 GPS 定位)
```text
T28386 AX+0.007 AY+0.026 AZ+0.978 GX+6.09 GY-1.05 GZ-2.80 P997.92 RH-0.1 KH-0.1 VZ+0.00 GA0.98 ST:0 MOD:E GPS:0,0 C:0
```

### 完整 ASCII 遙測字串 (GPS 定位成功)
當 GPS 成功定位時，字串中會額外附加 `LAT` (緯度) 與 `LON` (經度) 欄位：
```text
T28386 AX+0.007 AY+0.026 AZ+0.978 GX+6.09 GY-1.05 GZ-2.80 P997.92 RH-0.1 KH-0.1 VZ+0.00 GA0.98 ST:0 MOD:E GPS:1,8 C:0 LAT+25.04213 LON+121.53489
```

## 欄位詳細定義與 C 語言發送參考

| 欄位 Key | 資料型態 | 說明 | 格式範例 | C 語言 `sprintf` 格式參考 |
| :--- | :--- | :--- | :--- | :--- |
| **T** | `uint32` | 上電後時間 (單位：毫秒 ms) | `T28386` | `"T%lu"` |
| **AX** | `float` | 加速度 X 軸 (單位：g) | `AX+0.007` | `"AX%+0.3f"` |
| **AY** | `float` | 加速度 Y 軸 (單位：g) | `AY+0.026` | `"AY%+0.3f"` |
| **AZ** | `float` | 加速度 Z 軸 (單位：g) | `AZ+0.978` | `"AZ%+0.3f"` |
| **GX** | `float` | 角速度 X 軸 (單位：deg/s) | `GX+6.09` | `"GX%+0.2f"` |
| **GY** | `float` | 角速度 Y 軸 (單位：deg/s) | `GY-1.05` | `"GY%+0.2f"` |
| **GZ** | `float` | 角速度 Z 軸 (單位：deg/s) | `GZ-2.80` | `"GZ%+0.2f"` |
| **P** | `float` | Kalman 濾波後氣壓 (單位：hPa) | `P997.92` | `"P%.2f"` |
| **RH** | `float` | 相對起飛點高度 (單位：米 m) | `RH-0.1` | `"RH%.1f"` |
| **KH** | `float` | KF2 融合高度 (單位：米 m) | `KH-0.1` | `"KH%.1f"` |
| **VZ** | `float` | KF2 垂直速度 (單位：米/秒 m/s) | `VZ+0.00` | `"VZ%+0.2f"` |
| **GA** | `float` | 合加速度 (單位：g) | `GA0.98` | `"GA%.2f"` |
| **ST** | `int` / `str` | 飛行狀態代碼 (0=IDLE, 1=ARMED, 2=LAUNCH/LAUNCHED, 3=BOOST, 4=APOGEE, 5=DESCENT, 6=LANDED；亦支援對應大寫英文名稱) | `ST:0` 或 `ST:IDLE` | `"ST:%d"` 或 `"ST:%s"` |
| **MOD** | `char/hex`| 模組健康狀態 (1個十六進制字元表示 4-bit 狀態：BMP/IMU/LoRa/SD，皆正常為 `E` / 二進制 1110) | `MOD:E` | `"MOD:%X"` |
| **GPS** | `int,int` | 定位狀態與衛星數 (定位狀態：0=NO_FIX, 1=FIX；衛星數：0~12+) | `GPS:0,0` | `"GPS:%d,%d"` |
| **C** | `char/hex`| 發火迴圈導通狀態 (1個十六進制字元表示 4-bit 狀態：cond_A, cond_A_eff, cond_B, cond_B_eff) | `C:A` | `"C:%X"` |
| **LAT** | `float` | (選填) GPS 緯度 | `LAT+25.12345` | `"LAT%+0.5f"` |
| **LON** | `float` | (選填) GPS 經度 | `LON+121.56789` | `"LON%+0.5f"` |

> 💡 **移除的冗餘欄位**：
> - `RAW` (原始計數值，地面站不需要)
> - `BF` (狀態計數與內部緩衝調試值)
> - `PK` (最高高度，由地面站 GUI 自行動態計算)
> - `SD` (SD 寫入次數)
> - `LR` (LoRa 電訊號傳輸統計，由地面接收端自行計算掉包率即可)

## 火箭端特殊狀態/事件訊息格式 (Special Event/Log Format)

除了高頻傳輸的數值型姿態遙測數據外，火箭航電板可於重要事件發生時（如開傘觸發、氣囊充氣、感測器異常）傳送純文字狀態訊息。

### 訊息格式規範

格式由三個半形空格分隔的欄位組成，並以 `\r\n` 結尾：
```text
MSG <LEVEL> <CONTENT>\r\n
```

| 欄位順序 | 欄位名稱 | 型態/值 | 說明與作用 | 範例 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **Header (標頭)** | `MSG` | 固定文字標頭，表示此幀為火箭事件訊息而非遙測數值包 | `MSG` |
| **2** | **Level (等級)** | `INFO` / `WARN` / `ERROR` / `SUCCESS` | 訊息等級，主要用於判斷遠端開傘/氣囊驅動是否成功或系統異常 | `SUCCESS` |
| **3** | **Content (內容)** | `string` | 純文字訊息內容說明 (地面站採用 `maxsplit=2` 切割，內容中可包含任意半形空格) | `Parachute deployed successfully` |

> 💡 **文字空格說明**：地面站解析器僅切割前兩個半形空格以分離 Header 與 Level，第 3 個欄位 `Content` 會完全保留之後的所有內文與空格，不影響長句描述之完整性。

### 傳輸範例

* **開傘成功事件**：
  ```text
  MSG SUCCESS Parachute deployed successfully
  ```
* **開傘失敗/回路異常事件**：
  ```text
  MSG ERROR Parachute ejection circuit fault
  ```
* **氣囊充氣事件**：
  ```text
  MSG INFO Airbag inflation started
  ```

## 地面站對接說明

地面站解析模組採用**純 ASCII 直解析模式**（舊版 JSON 格式已完全廢棄，提升解析效能與反應速度）。
1.  **特殊訊息處理**：當收到以 `MSG` 開頭的數據列時，地面站會自動提取 Level 與 Content，並第一時間高亮印出至 Ground Station 控制台 Log 中。
2.  **3D 姿態估算**：當接收到新版遙測格式時，地面站會自動使用重力加速度估算公式計算 Roll 與 Pitch，因此地面站的 3D 立方體姿態指示器與線圖仍會照常運作。
3.  **異常警報**：若傳輸的資料損壞且不符合上述任何一種格式，地面站將於終端日誌中印出紅色警告提示。
4.  **角速度折線圖**：Z 軸角速度 (GZ) 已新增至地面站 GUI 的 Chart 3 (姿態與角速度圖表) 中。
